# Beneficiary Intelligence

## But
Suivre bénéficiaires, cohortes et parcours avec une perspective **impact + conformité + privacy**.

## Vues
- **Cohortes**: baseline/targets par cohorte + progression
- **Parcours**: participation aux activités/missions + evidence associée
- **Désagrégations**: genre, tranche d’âge, secteur, territoire
- **Qualité données**: complétude, incohérences, outliers (DQ)

## Privacy / Sécurité
- Minimisation (afficher le nécessaire selon rôle)
- Pseudonymisation possible pour reporting bailleur
- Audit complet des consultations/export (actor + motif)

## Offline UX
- consultation limitée (liste assignée)
- collecte d’attributs et preuve consentement (si applicable)

## Acceptance Criteria
- **AC-BEN-01**: désagrégations natives et filtrables (territoire/période).
- **AC-BEN-02**: export/consultation sensible auditée (purpose + actor).
- **AC-BEN-03**: compat offline pour collecte terrain avec sync différée.
